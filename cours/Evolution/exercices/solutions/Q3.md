# Critères d'évaluation

- Mentionner que vous devriez garder les vieilles méthodes en les rendant obsolètes avant de les enlever
- Suggérer que le coût de maintenance des vieilles méthodes est faible

# Solution suggérée

Comme la librairie est déjà utilisée hors de votre équipe, il est important de faire attention en changeant l'API.
Une bonne manière de faire cela est de rendre les vieilles méthodes obsolètes, pour donner du temps aux utilisateurs
de migrer vers les nouvelles méthodes.

De plus, les vieilles méthodes peuvent facilement être ré-implémentées par-dessus les nouvelles.
Il n'est donc pas nécessaire de les supprimmer le plus vite possible, elles pourraient même être gardées pour toujours.
